Zephyr build environment

Custom SP-1 firmware is a Zephyr RTOS application. Get Zephyr + its SDK installed, point a build at the stem_player board, and one command produces a flashable zephyr.bin. The sp1-midi BSP supplies the board definition so you don't start from scratch.

The build command that matters

From the sp1-midi repo root:

PATH="/path/to/zephyr-venv/bin:$PATH" \
ZEPHYR_BASE=/path/to/zephyrproject/zephyr \
ZEPHYR_SDK_INSTALL_DIR=/path/to/zephyr-sdk \
cmake -B build -GNinja -DBOARD=stem_player -DSNIPPET=cdc-acm-console

ninja -C build
ninja -C build flash   # SWD only — see below
  • -DBOARD=stem_player selects the board definition at boards/teenageengineering/stem_player/ — the device tree, pinctrl, and Kconfig defaults that describe the SP-1's hardware.
  • -DSNIPPET=cdc-acm-console adds a USB serial shell so you get log output during runtime (/dev/ttyACM0, prompt stem>). Strongly recommended for development — without it you're flying blind.
✓ No SWD? No problem (just slower)

ninja -C build flash needs an SWD probe. If you don't have one, just run ninja -C build to produce build/zephyr/zephyr.bin, then flash that file via the solderless web tool or your own bootloader client. Your loop becomes: edit → build → flash via bootloader → watch CDC logs → repeat. Slower than breakpoints, but it works.

What the BSP turns on (prj.conf highlights)

You inherit a config that already enables the right peripherals. The notable ones for an audio project:

# C++17 with full libc++
CONFIG_CPP=y
CONFIG_STD_CPP17=y

# Peripherals you'll need for audio
CONFIG_I2C=y                     # codec control
CONFIG_I2S=y                     # audio data bus
CONFIG_I2S_NRFX_TX_BLOCK_COUNT=8 # 8 DMA blocks, deep buffering
CONFIG_AUDIO=y
CONFIG_AUDIO_CODEC=y
CONFIG_DISK_ACCESS=y             # Zephyr disk subsystem for eMMC

# Input subsystem (faders + buttons)
CONFIG_INPUT=y
CONFIG_INPUT_ADC_KEYS=y          # analog button ladders
CONFIG_INPUT_LONGPRESS=y         # 3s long-press → power

# USB: composite MIDI 2.0 + CDC ACM
CONFIG_USB_DEVICE_STACK_NEXT=y   # required
CONFIG_USBD_MIDI2_CLASS=y
CONFIG_USBD_CDC_ACM_CLASS=y

# Performance — keep these for real-time audio
CONFIG_SPEED_OPTIMIZATIONS=y     # -O2
CONFIG_NRF_ENABLE_ICACHE=y
CONFIG_FPU=y
CONFIG_FPU_SHARING=y             # FPU usable across threads

Memory budget — you have lots of room

ResourceBare BSPFull TE firmware (for scale)
Flash~168 KB (19% of 892 KB slot)~264 KB
RAM~43 KB (17% of 256 KB)~219 KB (86%)

The bare MIDI BSP leaves ~720 KB of flash free. RAM is the tighter constraint once you add audio — the TE firmware spends most of its RAM on eMMC prefetch slots (~80 KB) and per-stem mixer + delay buffers.

Common build issues

BOARD not found
Run cmake from the repo root where boards/ lives.
STACK_NEXT missing
Your Zephyr base is too old; update it.
Enumeration fails
Host cached the old USB device — unplug/replug; on Linux reload udev rules.
Boots, no audio
Expected. The BSP is a MIDI controller; audio is something you add. → Audio engine
